# 10章 無関係の下位問題を抽出する
```
エンジニアリングとは、大きな問題を小さな問題に分割して、それぞれの解決策を組み立てることに他ならない。
```
■ 無関係な下位問題を抽出する手順
1. 関数やコードブロックを見て、高レベルの目標が何か考える
2. コードの各行に対して「高レベルの目標に直接的に効果があるのか？」「無関係の下位問題を解決しているのか？」を考える
3. 無関係の下位問題を解決しているコードが相当量あれば、それらを抽出して別の関数にする
